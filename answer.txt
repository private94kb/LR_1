1. Збережений (commited) - зміни є, але в локальній БД не входять; Змінений (modified) - модифіковані файли, при цьому помічені як готові для фіксації; Індексований (staged) -дані збережені в локальній БД.
2.Команду git add ми використовуємо для додавання будь-яких нових або змінених файлів в індекс.
3. Команда git status показує поточний стан файлів відносно моєї робочої області і репозиторія Git.
4. Команда git commit використовується для збереження змін у моєму локальному Git репозиторії.
5. Команда git init для підключення системи контролю версій до мого проекту (або папки).
6. У комітах потрібно писати опис зробленої роботи.
7. Команда git checkout використовується для переходу між гілками.
8. git reset використовується для відміни змін.
9. Команда git log використовується для переліку комітів зроблених в репозиторії в оберненому щодо хронологічного порядку.
10. Команда status показує поточний стан файлів відносно моєї робочої області і репозиторія Git.
11. Щоб відновити кілька файлів з попереднього коміта по імені викорістовується команда git reflog. 
13. git checkout -b (назва гілки) або git branch (назва гілки).
14. Щоб перейти на створену гілку git checkout (назва гілки) .
15. Щоб перейти на основну гілку git checkout master/main.
16. Щоб виконати злиття гілки вікористовуємо команду git merge.
